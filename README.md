It contains:

  * A‚ÄìC‚ÄìR‚ÄìP‚ÄìD parameters & submodules

* `case` ‚Äì uniform case schema:

## üß™ Quick Start: How to use the MIPractice model with an AI system

There are **two ways to work with the model** once the YAML file is loaded into an AI system:

### **1. Automatic Analysis**
The AI produces a complete **MIPractice_case** analysis of your thesis or scenario  
(A‚ÄìC‚ÄìR‚ÄìP‚ÄìD, IA-Box, A/M-band, trajectory, optional D-module).

Example:
> ‚ÄúAnalyse the following thesis using the MIPractice_case template:  
> *‚ÄòIn public institutions, symbolic actions often replace real responsibility.‚Äô*‚Äù

### **2. Interactive Reflection Mode**
The AI guides you **step by step**, asking targeted questions for each section.  
Your answers become part of the final analysis.

Example:
> ‚ÄúUse the MIPractice_case template in interactive mode.  
> Ask me the necessary questions for A‚ÄìC‚ÄìR‚ÄìP‚ÄìD, IA-Box, A/M-band and trajectory.‚Äù

---

### **How to start (3 simple steps)**

**Step 1 ‚Äî Load the YAML**  
Upload the file  
`MIPractice_case_v2.0_full_with_model_reference.yaml`  
and instruct the AI:

> ‚ÄúInternalise this YAML exactly as it is, without modification.‚Äù

This loads the full model (parameters, guardrails, IA-Box, scoring, case schema, etc.).

**Step 2 ‚Äî Provide a thesis or scenario**  
Give enough context for the model to work with, then choose automatic or interactive mode.

**Step 3 ‚Äî Read the structured output**  
The AI generates a full MIPractice_case analysis or a guided reflective process.  
The model intentionally surfaces missing information, ambiguities, tensions and alternative readings.

**Optional Step 4 ‚Äî Use the HTML Case Template for Clean Output**
You can produce a **polished, presentation-ready rendering** of your analysis using the HTML template:

```
example/Case Analysis Template.html
```

Ask the AI (after uploading the template):

> ‚ÄúTake the MIPractice_case analysis you just generated and insert the content into the HTML template from `Case Analysis Template.html`.
> Replace all placeholder fields (snapshot, ACRPD, IA-Box, A/M-band, findings, trajectory, conclusion).
> Return the completed HTML file.‚Äù

The template ensures:

* a **uniform professional layout**
* consistent structure across analyses
* ideal formatting for **workshops, leadership sessions, publications or screenshots**

It is fully optional ‚Äî but highly recommended when sharing or presenting results.
